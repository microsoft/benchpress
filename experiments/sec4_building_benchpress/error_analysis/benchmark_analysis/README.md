# §4.3 Benchmark-Side Prediction Error Analysis

## Paper mapping

- Section: `sec:reliability_analysis`, benchmark-side paragraph `sec:bench_predict`.
- Main table: `tab:predictability_factors`, generated by `gen_table.py`.
- Main figure: `fig:predictability_factors_51`, generated by `plot_section51.py`.
- Appendix metric-specific figures: generated by `plot_section51_appendix.py`.

## Purpose

This directory tests which benchmark-side factors make the fixed Logit Bias ALS point predictor easier or harder to use. H1--H3 are correlational hypotheses over benchmark-level features. H4--H7 are hide-half ablations that modify the training matrix and compare each treatment against a paired baseline. The former paired-sample-size diagnostic was retired because it asked the same scientific question as the stronger-neighbor support ablation.

## Inputs

- Canonical score matrix and metadata from the `benchpress` package.
- Official default BenchPress benchmark errors from `benchpress/evaluation/default_predictions/benchpress_default/by_benchmark.json` for H1--H3.
- H-specific ablation outputs for H4--H7.

## Outputs

- `H1_*`--`H3_*/results.json`: per-benchmark feature/error rows, Spearman tests, and raw prediction references where applicable.
- `H4_*`--`H7_*/ablation_results.json`: treatment records, Wilcoxon summaries, and raw base/treatment predictions.
- `tables/predictability_factors.tex`: generated table snippet.
- `figures/bp_predictability_factors_51.{pdf,png}`: main-text composite figure.
- `figures/bp_predictability_factors_51_<metric>.{pdf,png}`: metric-specific appendix composites.

## Run

```bash
cd ~/Documents/submission/benchpress/github/experiments/sec4_building_benchpress/error_analysis/benchmark_analysis
bash run.sh
```

For single-hypothesis debugging, run the corresponding `H*/analyze.py` or `H*/ablation.py` from this parent directory. H1--H3 can be regenerated cheaply; target-coverage and strong-neighbor-support ablations have dedicated README instructions for sharded remote runs.

## Resume / rerun

- H4 is resumable at the `(benchmark, seed)` shard-file level.
- H6 is resumable through `ablation_records_nested.jsonl`.
- To force a clean rerun, delete the relevant `results.json`, `ablation_results.json`, and any H-specific cache/shard files first.

## Interpretation

H1--H3 report Spearman correlations between a benchmark feature and prediction error. H4--H7 report paired Wilcoxon tests over one median treatment delta per benchmark. Main-text figures show the jointly supported effects; appendix figures provide the broader metric grid.
