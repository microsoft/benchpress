# Temporal Deployment Stress Test

## Paper mapping

Section 5.3 (Predicting Newly Released Models) in the main body, with the full protocol parked in Appendix C. Previously this experiment lived appendix-only; it was promoted to §5.3 because it answers the third operational question of Section 5 (deployment against a brand-new release).

## Purpose

Test the deployment setting where a new model appears after the current training matrix. For each landmark row, the predictor may use only models released before that row's cutoff date. For each target model in the row, `k` of that model's observed benchmark scores are revealed, and BenchPress predicts the remaining observed scores for that model. The shard records all observed scores of the active target models: revealed cells are exact zero-error predictions, hidden cells with finite BenchPress predictions enter MedAPE/MedAE, and hidden cells on benchmarks that have no finite deployment prediction are marked `not_predictable`. Rows such as GPT-4.1 family or Qwen 3 pool multiple related target variants only for reporting.

The paper-facing metrics are MedAPE and MedAE, computed with `benchpress.evaluation_harness.compute_prediction_error`.

## How to run

```bash
cd experiments/sec5_findings/temporal_deployment
bash run.sh
```

For a targeted shard:

```bash
python run.py --mode shard --family-key deepseek_r1 --k 10 --seed 0
```

After shards finish:

```bash
python run.py --mode merge
python gen_table.py
```

## Parallel execution

The independent unit is `(family, k, seed)`: 8 families x 6 k values x 10 seeds = 480 shards. `run.py --mode run-all --workers N` runs or reuses those shards in parallel. CPU thread counts should be pinned by `run.sh`; full runs should use a remote CPU machine.

## Inputs

- Canonical score matrix from `benchpress/build_benchmark_matrix`
- Model release dates from the canonical model records
- Current BenchPress default predictor: Logit Bias ALS, rank 2, lambda 0.1

## Outputs

- `results/shards/*.json.gz`: one shard per `(family, k, seed)`, including the target/training model IDs, per-target-model revealed benchmark IDs, per-cell raw rows for all observed target cells, and shard metrics. Raw rows include `is_revealed`, `is_metric_cell`, and `prediction_source`; revealed cells have `pred=actual`, while non-predictable hidden cells have `pred=null`.
- `results.json`: merged canonical payload with config, landmark definitions, raw predictions, and paper-facing summaries.
- `table.tex`: LaTeX table generated by `gen_table.py`.

## Resume / rerun

Existing shards are reused only if their config matches the requested `(family, k, seed)` and protocol version. Stale or incompatible shards are overwritten during `run-all`, while `--mode merge` refuses to merge incompatible or missing shards.
